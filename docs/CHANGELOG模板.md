## 更新指南

### 1. 确定变更类型

- **新增 (Added)**: 新功能、新接口、新文件、新模块
  - 例：新增用户设置页面
  - 例：appApi: 新增评论点赞接口 `appLikeComment`

- **变更 (Changed)**: 现有功能的修改、改进、参数调整
  - 例：优化首页加载速度
  - 例：appGetDynDetail: 返回数据中增加评论统计信息

- **修复 (Fixed)**: Bug修复、问题解决
  - 例：修复登录页面崩溃问题
  - 例：修复评论图片上传失败问题

- **移除 (Removed)**: 删除的功能、接口、文件
  - 例：移除旧版动态列表接口

- **弃用 (Deprecated)**: 即将移除但暂时保留的功能
  - 例：弃用 getUserInfo v1，请使用 v2

- **安全 (Security)**: 安全相关的修复或改进
  - 例：修复Token验证漏洞

- **待完成 (Pending)**: 计划中但未完成的功能
  - 例：帖子详情：评论列表与接口联调

### 2. 编写规范

- 使用中英文双语
- 每条变更一行
- 简洁明了，突出重点
- 按模块分类（如需要）
- 必要时添加接口名称或文件名

### 3. 示例

```markdown
## [未发布 / Unreleased]

### 新增 (Added)
- iOS: 评论列表支持一级/二级回复展示
- appApi: 新增 `appLikeComment` 评论点赞接口
- 文档: 评论功能完整接口文档

### 变更 (Changed)
- PostDetailView: 优化评论加载性能，支持分页加载
- appGetDynDetail: 返回数据中增加评论总数和最新评论
- 部署脚本: 优化云函数批量部署流程

### 修复 (Fixed)
- 修复评论图片上传后无法显示问题
- 修复评论点赞状态在刷新后丢失问题
- 修复二级回复数量显示错误问题

### 待完成 (Pending)
- 评论输入框: 添加@用户功能
- 评论: 支持富文本编辑
```

### 4. 发布新版本时

当准备发布新版本时：

1. 确定版本号（参考语义化版本）
   - Bug修复: 0.1.0 → 0.1.1
   - 新功能: 0.1.0 → 0.2.0
   - 重大更新: 0.1.0 → 1.0.0

2. 将 `[未发布 / Unreleased]` 改为版本号和日期
   ```markdown
   ## [0.2.0] - 2025-02-15
   ```

3. 在顶部创建新的 `[未发布 / Unreleased]` 部分

4. 检查并清理内容，确保描述准确完整

## 快速参考

### 变更类型速查

| 类型 | 使用场景 | 示例 |
|------|---------|------|
| Added | 新功能、新文件 | 新增评论功能 |
| Changed | 功能改进、修改 | 优化加载速度 |
| Fixed | Bug修复 | 修复崩溃问题 |
| Removed | 删除功能 | 移除旧版接口 |
| Deprecated | 即将移除 | 弃用v1接口 |
| Security | 安全修复 | 修复安全漏洞 |
| Pending | 未完成工作 | 待联调功能 |
