## 九、更新记录管理

### 9.1 CHANGELOG.md 维护规范

每次提交代码前，必须更新 `CHANGELOG.md` 文件，记录所有重要变更。

#### 更新时机
- **每次功能开发完成时**
- **每次Bug修复时**
- **每次重要配置修改时**
- **每次接口变更时**
- **准备发布新版本时**

#### 更新步骤

1. **在 [未发布 / Unreleased] 部分添加变更**
   ```markdown
   ## [未发布 / Unreleased]
   
   ### 新增 (Added)
   - 具体新增的功能描述
   
   ### 变更 (Changed)
   - 具体变更的功能描述
   
   ### 修复 (Fixed)
   - 具体修复的问题描述
   ```

2. **变更类型选择**
   - **新增 (Added)**: 新功能、新接口、新文件
   - **变更 (Changed)**: 现有功能改进、参数调整
   - **修复 (Fixed)**: Bug修复
   - **移除 (Removed)**: 删除的功能
   - **弃用 (Deprecated)**: 即将移除的功能
   - **安全 (Security)**: 安全修复
   - **待完成 (Pending)**: 未完成的工作

3. **发布新版本时**
   - 确定版本号（遵循语义化版本）
   - 将 Unreleased 内容移到新版本号下
   - 添加发布日期
   - 创建新的 Unreleased 部分

4. **版本号规则**
   - **修订号 +1** (如 0.1.0 → 0.1.1): Bug修复、小改进
   - **次版本号 +1** (如 0.1.0 → 0.2.0): 新功能添加
   - **主版本号 +1** (如 0.1.0 → 1.0.0): 重大功能更新或破坏性变更

#### 编写规范
- 使用中英文双语描述
- 每条变更一行，简洁明了
- 突出重点，便于快速浏览
- 相关模块变更可以分组描述
- 必要时添加链接或引用

#### 示例

```markdown
## [未发布 / Unreleased]

### 新增 (Added)
- appApi: 新增评论点赞接口 `appLikeComment`
- iOS: 评论列表视图支持二级回复展示
- 文档: 评论功能接口文档

### 变更 (Changed)
- appGetDynDetail: 返回数据中增加评论统计信息
- PostDetailView: 优化评论加载性能

### 修复 (Fixed)
- 修复评论图片上传失败问题
- 修复评论点赞状态不同步问题
```

### 9.2 与版本管理的关系

- **CHANGELOG.md**: 面向用户的更新记录，记录功能变更
- **版本与进度.md**: 项目管理文档，跟踪开发进度
- **日复盘.md**: 开发日志，记录每日工作
- **Git Commit**: 代码级变更记录

这四者相互配合，形成完整的项目历史追踪体系。
